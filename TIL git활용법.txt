<git 활용법>

git이란? 버전관리 도구입니다.
버전관리란? 변경된 사항을 기록하고, 언제 무엇이 어떻게 변경된지 알고 있는 것, 상태를 정리하는 것입니다.
각 버전의 프로젝트 상태 정보를 가지고 있으면 됩니다.

Commit : 변경된 프로젝트의 사항을 저장(포착)하는 것.
 - 커밋메세지는 작업내역이 어떤 것인지 알아볼 수 있게 적는 메시지.
   변경된 사항이 명료하고 누구나 보고 예상하기 쉽게 적어야 합니다.

Commit history : 변경된 사항이 저장된 곳입니다.

git 초기화(git initialize) : 컴퓨터에 있는 프로젝트를 Git 이 관리하는 프로젝트로 만들기

add (혹은 staging, 스테이징) : commit 에 반영할지 안할지는 파일 단위로 선택 가능. commit 에 반영할 파일을 선택하는 것

repo : 'Git으로 관리되는 프로젝트' 를 Git 에서는 repo(리포, repository 리포지토리의 약자) 라고 부름. 

 - 내 컴퓨터에 저장되어있는 리포지토리를 로컬 repo(local repository)

 - Github 처럼 다른 곳에서 접속할 수 있는 공간에 저장되어있는 것을 원격 repo(remote repository)

Trading : 원격리포, 로컬리포를 연결시키는 것.

Push : 로컬리포 -> 원격리포

Pull : 원격리포 -> 로컬리포

Clone : 원격리포를 초기에 다운로드하는 것.

프로젝트 파일에.git이 생기면 : 기존 프로젝트에 git프로젝트라는 속성이 하나 추가된것.
